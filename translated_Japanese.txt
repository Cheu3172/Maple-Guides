// デバッグ // チュウによるセットアップ // 2022年

このプロセスを開始する前の注意。
- このプロセスはDefender Controlのようなソフトウェアのためのものではありません。
サードパーティ製のソフトウェアで無効にしている場合は、ウィンドウズ・ディフェンダーの再有効化をお試しください。また、サードパーティ製のソフトウェアでウィンドウズディフェンダーを無効にし ていない場合は、そのようなことをしないようにしてください。

---------------------------------------------
通常のコンピュータのセットアップ

ウィンドウズ・ディフェンダー
1. ウイルスと脅威の保護」の設定をすべて無効にする。
2. Exploit Protection の設定をすべて無効にする
3. コアアイソレーションのすべての設定を無効にする

オーバーレイ
1. Discord オーバーレイを無効にする
2. GeForce Expierenceオーバーレイを無効にする
3. Overwolf Overlayを無効にする - [クラッシュの原因になることが知られています].

アンチチートとサードパーティアンチウイルス
1. Vanguardを無効にする/アンインストールする
2. サードパーティーのアンチウィルスを無効にするか、Maple.Software (Loader)の除外を追加してください。
[Kasperskyなどのアンチウィルスは除外が効かないため、完全に無効化する必要があります]。

---------------------------------------------
コンピュータのセットアップを変更する (通常のコンピュータのセットアップからオーバーレイとアンチチート // サードパーティAVをフォローする)

Window's Defender と Exploit Protection がデフォルトでシステムから削除されている場合、以下のステップを正確に実行してください。

エクスプロイト・プロテクション
1. adminstrator privでCMDを開く
2. 2. bcdedit.exe /set {current} nx AlwaysOff を貼り付けて実行します。
3. ポリシーエディタを開く
4. Local Computer Policy (ローカルコンピュータポリシー), Computer Configuration (コンピュータ構成), Administrative Templates (管理テンプレート), System (システム), Device Guard (デバイスガード) に移動する。
5. リスト内のすべての設定を無効にする
6. コンピュータを再起動する

Key-Registery
1. ComputerHKEY_LOCAL_MACHINE ⇄SYSTEM ⇄CurrentControlSet ⒸDeviceGuardScenarios
HypervisorEnforcedCodeIntegrityとしてDWORDキーを作成し、値を0に設定します。
3. リスタート

Mapleのデバッグ(クラッシュ)
最初のチェック： https://maple.software/help/software-issues あなたの問題に対する解決策が見つからない場合は、以下のステップに従います...
第二に、もし "Infinite loading"、"Doesn't inject"、"Crashes on inject "であれば、上記のステップを正しく踏んでいないことになります！以下の情報を提供してください。

次のことを教えてください。

詳細
1. クラッシュの詳細をできるだけ詳しく教えてください。
2. 可能であれば、エラーコードのスクリーンショット（表示されないこともあります）。

イベントビューア
1. イベントビューア」を検索して、開いてください。
2. Windowsログ "から "アプリケーション "に移動します。
3. そこから、赤丸の「！」で示された最新のクラッシュを提供し、その一般的な詳細をコピーし（何も変更しないでください）、レポートの横に貼り付けてください。
注意：Mapleがイベントビューアのログを生成しない場合があります。その場合は、このステップをスキップしてください。

大須！ランタイム
1. Osu!Runtimeを開きます。フォルダを開き、"Logs "に移動します。
2. Runtime」をDiscordにドラッグ＆ドロップし、レポートと一緒に投稿してください。

メイプルランタイム
1. 検索バーで "実行 "と入力する（このプロセスではosu!）
2. 2. "%appdata%"と入力し、ランダムなテキストの束がある大きなフォルダに移動します。
3. そして、"Logs "に移動し、そこで "Runtime "を見つけてください。そのファイルをあなたのレポートと一緒にdiscordにドラッグ＆ドロップしてください。

記録
1. 可能であれば、レポートと一緒に録音を提供し、クラッシュを再現してください。
2. 2. Maple が他のプログラムと一緒にクラッシュし、そのプログラムが Github (オープンソース) である場合、レポートと一緒にそれを提供してください。
